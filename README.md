# Структура проекта для `aiogram`

## Шаги для ее инициализации

### 1 Шаг. Склонировать репо
```commandline
git clone ...
```

### 2 Шаг. Установить виртуальное окружение
```commandline
python3 -m venv .venv
```

### 3 Шаг. Активировать виртуальное окружение
- Windows
```commandline
.venv\Scripts\activate.bat
```
- MacOS/Linux (Bash)
```bash
source .venv/bin/activate
```

### 4 Шаг. Установить начальные, необходимые зависимости
```commandline
pip install -U aiogram python-dotenv watchfiles
```

### 5 Шаг. Создать файл .env
```bash
echo "BOT_TOKEN=ТОКЕН_ТВОЕГО_БОТА" > .env
```

### 6 Шаг. Запустить программу
```commandline
watchfiles "python main.py" .
```

## Работа с бронированиями

В проекте реализованы функции для управления бронированиями пользователей.

### `create_reservation()`

Создает новое бронирование пользователя.

**Параметры:**

- `user_id` — идентификатор пользователя.
- `table_id` — идентификатор выбранного стола.
- `date` — дата бронирования.
- `time` — время бронирования.
- `guests` — количество гостей.

**Как работает:**

1. Создается новое подключение к базе данных.
2. Формируется объект `Reservation`.
3. Бронированию автоматически устанавливается статус `"active"`.
4. Данные сохраняются в базу данных.
5. Функция возвращает созданное бронирование.

---

### `get_user_reservations()`

Получает список всех бронирований конкретного пользователя.

**Параметры:**

- `user_id` — идентификатор пользователя.

**Как работает:**

1. Выполняется поиск записей в таблице бронирований.
2. Выбираются только бронирования указанного пользователя.
3. Возвращается список найденных бронирований.

Если у пользователя нет бронирований, возвращается пустой список.

---

### `delete_reservation()`

Удаляет бронирование по его идентификатору.

**Параметры:**

- `reservation_id` — идентификатор бронирования.

**Как работает:**

1. Поиск бронирования в базе данных по ID.
2. Если запись найдена — она удаляется.
3. Изменения сохраняются в базе данных.

**Возвращаемое значение:**

- `True` — бронирование успешно удалено.
- `False` — бронирование с таким ID не найдено.
